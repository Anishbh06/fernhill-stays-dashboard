# Fernhill Stays — Performance Dashboard

A data-driven dashboard for Fernhill Stays, a boutique hotel group operating 5 properties in Bengaluru. Built to answer three questions: how each property is doing, which booking channels are worth it, and where to focus effort (health score).

## Live Dashboard

🔗 **[View Live Dashboard](https://fernhill-stays-dashboard-yrnwtdbmxdykhk5wlthrkm.streamlit.app/)**

## Quick Start (Local)

```bash
# 1. Clone the repo
git clone https://github.com/YOUR_USERNAME/fernhill-stays-dashboard.git
cd fernhill-stays-dashboard

# 2. Install dependencies
pip install -r requirements.txt

# 3. Run the data cleaning pipeline
python clean_data.py

# 4. Launch the dashboard
streamlit run app.py
```

The dashboard will open at `http://localhost:8501`.

## What the Dashboard Shows

1. **Property Performance** — Revenue, bookings, cancellation rates, and monthly trends per property
2. **Channel Analysis** — Which booking channels deliver the most value (revenue, cancellation rate, avg booking value)
3. **Health Score** — Composite 0–100 score per property (30% occupancy, 25% revenue, 25% cancellation, 20% rate)

## Data Cleaning Summary

The raw dataset had 238 rows with 14 distinct issue types. The cleaning pipeline (`clean_data.py`) handles:
- 8 duplicate rows removed
- 11 amount errors corrected (negative values, 10x typos)
- 85 cancelled/no-show bookings excluded from revenue
- Text normalisation across 4 columns (9 property variants → 5, etc.)
- 4 date formats parsed + 20 day/month swaps auto-corrected
- Missing values flagged (not imputed)

See [DECISIONS.md](DECISIONS.md) for full details.
